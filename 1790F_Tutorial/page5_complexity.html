Page 5: Time & Space Complexity

Space Complexity: O(N)

We need to store the structure of the tree and a few state arrays:

  • The adjacency list adj stores an undirected tree, requiring 2N - 2 integers.
  • The c array (sequence of black nodes) takes O(N) space.
  • The min_dist_to_black array takes O(N) space.
  • The BFS queue takes at most O(N) space in the worst case.

We don't use any large matrices O(N²) or heavy deep recursion stacks, making it strictly linear O(N) space.

Time Complexity: O(N log N) / O(N√N)

The worst-case time complexity without pruning would be O(N²) (running a full BFS from N nodes). However, thanks to the pruning condition if (min_dist_to_black[u] >= ans) continue;, the time complexity is amortized and heavily bounded.

Why it passes the time limit:

  1. Every time we add a new black node, the global ans can only decrease.
  2. When we run BFS, we ONLY explore neighboring nodes up to a depth of ans.
  3. Because ans rapidly drops as we add more black nodes, the "exploration radius" of our BFS shrinks significantly very quickly.
  4. If a cluster of nodes is close together, ans drops to a tiny number (like 1 or 2) almost immediately, meaning all future BFS runs will stop instantly!
