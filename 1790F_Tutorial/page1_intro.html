Page 1: Tree Setup & The Infinity Value

Building the Tree

In our code, we read the tree's connections and build an undirected graph using an adjacency list:

int u, v; cin >> u >> v;
adj[u].pb(v);
adj[v].pb(u);

Because the tree paths go both ways (undirected), we must store v in u's list, AND store u in v's list. If we didn't do this, our Breadth-First Search (BFS) would only be able to travel one way!

Why initialize with n + 5?

We use an array called min_dist_to_black to track the shortest distance from any node to any black node.

vector<int> min_dist_to_black(n + 1, n + 5);
int ans = n + 5;

We initialize it with n + 5 as a safe "infinity" value. In a tree with n vertices, the maximum possible distance between any two nodes is n - 1. By setting it to n + 5, we guarantee it is strictly greater than any valid shortest path.

We avoid INT_MAX because adding 1 to it later could cause integer overflow!
